# Internationalization rules

ทุก text ที่ user เห็นต้องอยู่ในรูปแบบที่เปลี่ยนภาษาได้ — แม้ตอนนี้รองรับแค่ EN+TH

## Bilingual labels

ทุก UI string ที่ user เห็นต้อง:
1. อยู่ใน `src/lib/i18n/labels.ts` — ห้ามฮาร์ดโค้ดในคอมโพเนนต์
2. มีทั้ง EN + TH

```ts
// src/lib/i18n/labels.ts
export const labels = {
  blog: { en: 'Journal', th: 'บทความ' },
  works: { en: 'Works', th: 'ผลงาน' },
  about: { en: 'About', th: 'เกี่ยวกับเรา' },
  inquireProject: { en: 'Start a project', th: 'เริ่มโปรเจกต์' },
  featuredWork: { en: 'Featured work', th: 'ผลงานเด่น' },
} as const;
```

ใช้ใน component:

```tsx
import { labels } from '@/lib/i18n/labels';
import { useLocale } from '@/lib/i18n/useLocale';

const lang = useLocale();   // 'en' | 'th'
<button>{labels.addToBag[lang]}</button>
```

## Bilingual display pattern (จาก design)

design ใช้ pattern `English · ภาษาไทย` ในบาง chrome (tab bar, header) — แสดงทั้ง 2 ภาษาพร้อมกัน:

```tsx
<span>Home · หน้าแรก</span>
<span>Works · ผลงาน</span>
```

ใช้เมื่อ:
- Tab labels (พื้นที่จำกัด, ทั้ง 2 ต้องสั้น)
- Major section headings ที่อยากให้แบรนด์ดู bilingual

ห้ามใช้กับ body content ยาวๆ (จะอ่านยาก) — เลือกภาษาเดียวตาม locale

## Where bilingual matters most

| Content | Strategy |
|---|---|
| Nav, button, label, error message | `labels.ts` lookup |
| Product name, description | DB เก็บ 2 column: `name_en`, `name_th` (ทำใน V2) — V1 = ภาษาเดียว default EN |
| Date / number | ใช้ `Intl.DateTimeFormat`, `Intl.NumberFormat` |
| Currency | THB ตลอด V1 — display `฿` หรือ `THB` ตาม locale |

## Date/number formatting

ห้าม hard-format `1,000.00`, `2024-01-15` — ใช้ Intl:

```ts
new Intl.NumberFormat('th-TH', { style: 'currency', currency: 'THB' }).format(1234.5);
// → '฿1,234.50'

new Intl.DateTimeFormat('th-TH', { dateStyle: 'long' }).format(new Date());
// → '15 มกราคม 2569' (ปี พ.ศ.)
```

ระวัง: ปี พ.ศ. — ลูกค้าไทยคุ้น แต่ EN audience ต้อง `'en-US'` locale สำหรับปี ค.ศ.

## Pluralization

ใช้ `Intl.PluralRules` หรือ ICU MessageFormat ถ้ายาก — V1 ทำง่ายๆ:

```ts
const itemLabel = (n: number, lang: 'en' | 'th') => {
  if (lang === 'th') return `${n} รายการ`;
  return `${n} ${n === 1 ? 'item' : 'items'}`;
};
```

(ภาษาไทยไม่มี plural form — ตัวเลข + คำลักษณนาม)

## Locale persistence

- เก็บ user preference ใน cookie `locale` (`en` | `th`)
- Server: อ่านผ่าน `cookies()` ใน RSC
- Client: provide ผ่าน Context จาก root layout

## Right-to-left

ปัจจุบันยังไม่รองรับ RTL — แต่ทุก layout ใช้ Tailwind logical properties ถ้าได้ (`ms-` แทน `ml-`,
`me-` แทน `mr-`) — เผื่อ V3 รองรับภาษา RTL

## Files vs DB strings

- Static UI text → `labels.ts`
- Dynamic content (product, category, post) → DB column
- Mixed (e.g., `quantityEstimate` validation message) → zod schema (ดู rules/validation.md — Thai messages)

## Adding a new label

1. เพิ่ม key ใน `labels.ts` — ทั้ง `en` + `th` พร้อมกัน
2. ห้าม commit key ที่ขาดภาษาใดภาษาหนึ่ง — TS type จะไม่ผ่าน (object literal must satisfy type)
3. ตรวจว่า component ใช้ลายงาน label นี้ — ห้าม dead key

```ts
type Label = { en: string; th: string };
type Labels = Record<string, Label>;
export const labels: Labels = { /* ... */ };
```
